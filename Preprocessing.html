How do we implement preprocessing (client-side vs server-side)?

   Preprocessing is transforming raw data into an understandable format, for the processor to compile, through data mining. The way to implement data preprocessing is to import libraries, then the dataset, handling of missing data, then of categorical data, split dataset into testing and training sets, and finally scaling.

What are some of the mechanisms used in either LESS or SASS?

   In LESS, you can have cleaner structures with nesting, create variables to use for layout options and mixins. Nesting allows the creator of the webpage to incorporate like values together under one set of brackets. This way you can more easily find a section to make changes if the coding doesn't seem to be working out like you thought. Creating variables and reusing whole classes comes in handy as well. This also ties in mixins and what they are used for. By creating a variable like @main-color, you can then use that color variable throughout your coding by calling on it and also be able to simply change the color when you do. Mixins are just the reusable classes that contain the variable(s) that are needed for that particular section of coding.
